# A Multi-Objective Spatial Optimization Framework for Climate Justice

### 城市冷却服务空间优化与气候正义评价框架

## 📖 框架概述

本框架提供了一套集成 **GIS 空间分析** 与 **多目标进化算法（NSGA-II）** 的标准化工作流，旨在解决城市公共服务资源分配失衡的问题。框架核心逻辑是在有限的财政预算与空间约束下，通过**“逆向修正”**机制平衡系统效率（Efficiency）与分配公平（Equity），将资源精准引导至服务水平最低且社会脆弱性最高的区域。

---

## 🛠 五阶段标准化工作流

### 第一阶段：多源数据整合 (Universal Data Integration)

构建三维一体的基础数据库，涵盖以下通用维度：

1. **城市地理空间数据 (Geo-Spatial)**：包含城市行政边界、街道网络数据集（用于路径分析）及建筑轮廓数据。
2. **基础设施数据 (Infrastructure)**：
* **存量设施**：已有的集中式或固定公共服务节点及其服务能力参数。
* **待选设施类型**：拟新增的分布式或微型服务设施，需具备明确的效能系数与成本等级。


3. **社会经济与环境指标 (Socio-Env Indicators)**：反映人群脆弱性的多维指标（如收入水平、人口密度）及反映环境压力的物理数据。

### 第二阶段：空间现状诊断与优先级筛选

1. **基线可达性测算 ($A_{baseline}$)**：利用网络分析工具，基于实际路网计算服务设施的步行/行车距离，生成精细化空间单元的服务可达性分布。
2. **服务盲区识别 (Gap Detection)**：根据设施类型设定分级服务半径（缓冲区），通过空间叠加分析识别不受现有设施覆盖的“绝对服务真空区”，并将其定义为算法优化的最高优先级区域。
3. **干预区域筛选 (Target Triage)**：根据“高需求-低服务”的双重标准，从全域样本中筛选出最亟需干预的核心子集。
4. **微观落位约束 (Siting Logic)**：建立基于地理特征的落点准则，确保新增设施点位落在合法的物理空间（如路网边线、公共开敞空间）内。

### 第三阶段：多目标优化寻优计算 (NSGA-II Computation)

1. **双目标函数设定**：
* **目标 1 (公平导向 $Z_1$)**：最大化服务缺口修复收益，通过对低基线、高需求区域进行权重补偿，迫使资源流向“服务孤岛”。
* **目标 2 (效率导向 $Z_2$)**：最大化单位资源的覆盖效能，优化全域服务总量的边际增益。


2. **约束条件设定**：
* **财政预算约束**：设定总成本或设施总数的硬性上限。
* **空间承载约束**：设定单一分析单元内设施投放的物理容积上限。


3. **解集筛选**：运行得出帕累托（Pareto）前沿解集，并采用决策支持算法（如最小曼哈顿距离法）识别最优折衷方案。

### 第四阶段：方案落地与效能评价

1. **指标重构**：将优化后的空间落点回传至地理信息系统，重构干预后的全域可达性图谱。
2. **效能评估**：
* **离散度分析 (CV)**：验证优化方案对全域服务失衡程度的修正作用。
* **缺口闭合率 (Gap Closure)**：量化新增投入对原始服务真空的填充效能。
* **相对增益分析**：对比各区域在优化前后的可达性提升率。


### 第五阶段：统计稳健性与敏感性分析

1. **参数敏感性测试**：调整不同类型设施的权重或校准系数，观察在不同投资偏好下系统效能的波动情况。
2. **逻辑稳健性检验**：
* 通过 **Pearson 相关性分析 ($r, p$)** 验证分配逻辑。
* **核心标准**：新增资源配给量必须与基线可达性呈现显著的**负相关关系**，以从数理层面证实算法具备“逆向纠偏”能力，确保框架始终遵循“按需分配”的公平原则。

## 🌟 框架优势

* **普适性**：不限制设施类型，可广泛应用于能源网络、冷却设施、医疗点等。
* **决策透明化**：将复杂的规划权衡从主观经验转向可量化、可追溯的算法模型。
* **底线保障**：在财政资源稀缺的背景下，通过差异化配给策略强制抬升最脆弱区域的民生防御底线。

## 🛠 工作流与技术细节

### 第一阶段：多源数据获取 (Data Acquisition)

框架需要以下三类核心数据，以支撑社会-物理-设施的综合评价：

1. **Urban Geo-Spatial (城市地理空间)**：行政边界 (NTA/CDTA)、LION 街道底图（用于设施落位约束）、建筑轮廓数据 (Building Footprint)。
2. **Infrastructure (基础设施)**：
* **固定大型设施**：冷却中心 (Cooling Centers)、避难所 (Shelters)、公园泳池 (Pools)。
* **分布式微型设施**：喷雾淋浴 (Spray Showers, SS)、饮水喷泉 (Drinking Fountains, DF)。


3. **Socioeconomic & Thermal (社会经济与热环境)**：热脆弱性指数 (HVI)、社会脆弱性指数 (SVI)、地表温度 (LST) 及低收入人口分布。

### 第二阶段：ArcGIS Pro 预处理 (Spatial Pre-processing)

在运行优化算法前，需在 ArcGIS Pro 中完成以下操作：

1. **综合可达性计算 ($A_i$)**：利用 Network Analyst 计算步行可达性，并根据社区尺度进行分级色彩可视化。
2. **绝对盲区识别 (Blind Spot Detection)**：
* 建立分级缓冲区：CC (500m)、SS (200m)、DF (100m)。
* 使用 **Erase (擦除)** 工具剔除覆盖区，识别出冷却服务的“真空地带”，作为算法优化的最高优先级区域。


3. **落位逻辑约束 (Siting Logic)**：
* 将新增点位约束在 **LION 路网数据集** 上。
* 避开建筑区域，确保设施在物理空间上的可达性与实操性。



### 第三阶段：NSGA-II 多目标优化 (Algorithm Core)

使用提供的 Python 代码运行优化。代码严格遵循以下逻辑：

1. **目标函数设定**：
* **$Z_1$ (Equity)**：最大化可达性缺口修复收益，公式：$\sum (Gap_i^2 \times Low\_Income\_Pop \times \Delta A_i)$。
* **$Z_2$ (Efficiency)**：最大化全域综合冷却效能，公式：$\alpha \sum \Delta A_i - \beta \sum(x_{ss} + x_{df})$。


2. **参数校准**：
* **提升系数**：$\Delta A_{SS} = 0.00018, \Delta A_{DF} = 0.00015$。
* **成本权重**：SS 设为 2，DF 设为 1（对应其实际财政成本差异）。


3. **约束条件**：
* **物理上限**：单个社区新增设施总数 $\le 10$。
* **底线保障**：对于 $A_i < 3.0$ 且低收入人口 $> 15,000$ 的社区，强制分配 $\ge 1$ 个设施。

### 第四阶段：方案筛选与验证 (Evaluation)

1. **最优解筛选**：采用 **最小曼哈顿距离法 (Minimum Manhattan Distance)** 从 Pareto 前沿中识别最优折衷解 (Solution 2)。
2. **统计显著性验证**：
* **Pearson 相关性**：验证新增资源量与基线可达性之间的关系。框架要求 **$r < -0.2$ 且 $p < 0.05$**，以证实“逆向修正”机制的有效性。
* **多样性验证**：通过“种群拥挤距离 (Crowding Distance)”监控算法在进化过程中对解集多样性的维持能力。



### 第五阶段：敏感性分析 (Sensitivity Analysis)

* 通过调节设施类型权重，观察在固定预算（如 600 万美元）下，哪种设施配比能使全域冷却服务增益最大化。
* 验证不同权重场景下，资源分配是否始终保持向低可达性区域倾斜的鲁棒性。
